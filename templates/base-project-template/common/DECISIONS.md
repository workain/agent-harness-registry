# Decisions

Append-only log of **why**, in date order — not what changed (your project's own history —
git log or otherwise — already has that). One entry per decision that would otherwise only live
in someone's head or a closed conversation. Cheap to maintain; this is one of the two artifacts
(alongside a per-task running log) with the strongest evidence of actually being used, across
every scale this pattern has been observed at.

## Format

```
## YYYY-MM-DD — <decision, one line>
<1-3 sentences: what was decided and why. Link the issue/PR/conversation if one exists.>
```
