# Notes

Flat, general-knowledge scratch file: things worth recording that aren't a decision (that's
`DECISIONS.md`) and aren't a personal/session lesson (that's `LESSONS.md` or your runtime's
memory feature) — reference material, research findings, external best-practice notes,
whatever genre of "things this project knows" comes up first.

**Split this into subdirectories only once a genre of content actually accumulates** — e.g.
once you have real literature notes AND real landscape-survey notes AND real distilled
methodology as three genuinely distinct, separately-growing bodies of content, that's the
signal to split into `knowledge/{papers,landscape,methods}/` or whatever taxonomy fits. Doing
that split on day one, before any content exists to justify it, is empty scaffolding — a single
flat file that's actually written to outperforms an elaborate directory structure that isn't.
